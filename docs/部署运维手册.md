# 智瞳 Aura-PHM 部署与运维指南

---

## 一、部署环境要求

### 1.1 硬件要求

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | Intel i5 8代 / AMD Ryzen 5 2600 | Intel i7 10代+ / AMD Ryzen 7 |
| 内存 | 8 GB | 16 GB |
| 磁盘空间 | 2 GB（安装 + 运行时） | 5 GB（含日志和缓存） |
| 显卡 | 集成显卡 | 独立显卡（3D 大屏性能更佳） |
| 显示器 | 1920×1080 | 2560×1440（大屏展示） |

### 1.2 软件环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 x64 (1903+) / Windows 11 |
| .NET Framework | 4.7.2+（NSIS 安装器依赖） |
| Visual C++ Redistributable | 2019+ |
| Python | 3.8+（安全工具可选） |
| 网络 | 可选（离线模式全功能可用） |

### 1.3 网络环境（在线模式）

| 用途 | 目标地址 | 端口 | 协议 |
|------|----------|------|------|
| Claude API | api.anthropic.com | 443 | HTTPS |
| 第三方中转 | 自定义 baseURL | 443 | HTTPS |
| 本地 Bridge | 127.0.0.1 | 17654 | HTTP |

## 二、安装部署

### 2.1 安装版（NSIS Installer）

1. 双击 `AI预测性维护演示平台-2.0.0-Setup.exe`
2. 选择安装路径（默认 `C:\Program Files\AI预测性维护演示平台`）
3. 选择是否创建桌面快捷方式
4. 点击「安装」等待完成
5. 勾选「运行程序」完成安装

**卸载**：控制面板 → 程序 → 卸载，或运行安装目录下 `Uninstall AI预测性维护演示平台.exe`

### 2.2 便携版（Portable）

1. 将 `AI预测性维护演示平台-2.0.0-Portable.exe` 放置到任意目录
2. 双击运行
3. 首次运行会在同目录生成配置文件夹

**注意**：便携版运行后会在 `%APPDATA%` 下生成 Electron 用户数据目录。

### 2.3 开发环境部署

```bash
# 1. 克隆代码
git clone <repository-url>
cd AI设备预测性维护

# 2. 安装依赖
npm install

# 3. 开发模式启动（HMR 热更新）
npm run dev

# 4. 类型检查
npm run typecheck

# 5. 生产构建
npm run build

# 6. 打包分发
npm run dist
```

## 三、配置管理

### 3.1 环境变量配置

创建 `.env` 文件于项目根目录（开发模式）或应用安装目录（生产模式）：

```env
# AI 运维助手配置
ANTHROPIC_API_KEY=sk-ant-api03-xxxxxxxxxxxx
ANTHROPIC_BASE_URL=https://api.anthropic.com
ANTHROPIC_MODEL=claude-opus-4-8

# 浏览器插件 Bridge 认证
ELECTRON_BRIDGE_TOKEN=your-secret-token
```

### 3.2 配置说明

| 配置项 | 默认行为 | 配置后行为 |
|--------|----------|------------|
| ANTHROPIC_API_KEY 未设置 | 离线演示模式 | 在线 AI 推理 |
| ANTHROPIC_BASE_URL 未设置 | 直连 Anthropic | 通过中转服务调用 |
| ELECTRON_BRIDGE_TOKEN 未设置 | Bridge 无鉴权 | 需携带 Token 才能推送 |

### 3.3 第三方中转配置

如需使用国内中转服务（解决网络访问问题）：

```env
ANTHROPIC_BASE_URL=https://your-relay-provider.com/v1
ANTHROPIC_API_KEY=your-relay-key
```

要求中转服务兼容 Anthropic Messages API v1 格式。

## 四、功能模块运维

### 4.1 仪表盘模块

| 检查项 | 正常表现 | 异常处理 |
|--------|----------|----------|
| KPI 卡片数据 | 5 个卡片全部显示数值 | 重启应用 |
| 趋势图加载 | 显示 6 条设备时序线 | 检查数据源 |
| 告警表显示 | 彩色等级标签 + 完整文本 | 检查 CSS 加载 |
| 维护记录表 | 完整显示所有记录 | 刷新页面 |

### 4.2 AI 运维助手

| 状态 | 表现 | 处理方式 |
|------|------|----------|
| 在线正常 | 流式输出，显示 token 用量 | — |
| API Key 无效 | 提示认证错误 | 检查 .env 中 Key 是否正确 |
| 网络超时 | 显示错误后自动降级 | 检查网络/中转服务状态 |
| 离线模式 | 底部显示"离线演示模式" | 预期行为，无需处理 |
| 频率限制 | RateLimitError | 等待后重试或降低调用频率 |

### 4.3 TADPE 引擎

| 状态 | 表现 | 说明 |
|------|------|------|
| 空闲 | 所有指标为初始值 | 等待启动 |
| 运行中 | 5 阶段依次推进（~14秒） | 正常模拟过程 |
| 完成 | 所有指标最终收敛 | 可再次启动 |
| 热力图异常 | 启动后热力图消失 | 已在 v2.0.0 修复 |

### 4.4 数字孪生大屏

| 检查项 | 说明 |
|--------|------|
| 3D 渲染性能 | 如帧率低于 15fps，检查 GPU 驱动 |
| WebGL 兼容性 | 需要支持 WebGL 2.0 的 GPU |
| 粒子动画 | Canvas 粒子渲染，CPU 绑定 |
| 数据刷新 | 3 秒自动刷新周期 |

### 4.5 网络态势感知

| 功能 | 说明 |
|------|------|
| 实时指标 | Mock 数据 3 秒刷新 |
| 安全工具 | 需要 Python 3.8+ 和 rich 包 |
| 终端输出 | 模拟执行，非实际扫描 |

### 4.6 浏览器插件

| 检查项 | 处理方式 |
|--------|----------|
| 插件无法连接 | 确认桌面端已启动（Bridge :17654） |
| 推送无响应 | 检查 Token 是否匹配 |
| 跨域错误 | Bridge 已配置 CORS，检查浏览器安全策略 |

## 五、性能监控

### 5.1 关键性能指标

| 指标 | 目标值 | 监控方式 |
|------|--------|----------|
| 首屏加载 | ≤ 2 秒 | DevTools Performance |
| 内存占用 | ≤ 512 MB | 任务管理器 |
| CPU（空闲） | ≤ 5% | 任务管理器 |
| CPU（引擎运行） | ≤ 30% | 任务管理器 |
| 2D 图表帧率 | ≥ 30fps | Performance Monitor |
| 3D 大屏帧率 | ≥ 20fps | Performance Monitor |

### 5.2 性能优化建议

| 场景 | 优化方式 |
|------|----------|
| 内存占用高 | 关闭不使用的视图标签页 |
| 大屏帧率低 | 更新 GPU 驱动 / 降低粒子密度 |
| AI 响应慢 | 使用更近的中转节点 / 切换更快模型 |
| 启动时间长 | 确保 SSD 磁盘 / 关闭杀毒软件实时扫描 |

## 六、日志与故障排查

### 6.1 日志位置

| 日志类型 | 路径 |
|----------|------|
| Electron 主进程 | `%APPDATA%/ai-predictive-maintenance/logs/main.log` |
| 渲染进程 | DevTools Console (Ctrl+Shift+I) |
| 构建日志 | 项目根目录 `out/` |

### 6.2 常见故障排查

#### 启动后白屏

```
原因：渲染进程加载失败
排查：
1. Ctrl+Shift+I 打开 DevTools 查看 Console 错误
2. 检查 out/renderer/ 目录是否存在 index.html
3. 确认 Node.js >= 18
4. 尝试 npm run build 查看编译错误
```

#### AI 助手无响应

```
原因：IPC 通信或 API 调用异常
排查：
1. 检查 .env 文件格式是否正确（无多余空格）
2. 在终端测试 API Key: curl -H "x-api-key: YOUR_KEY" https://api.anthropic.com/v1/messages
3. 如使用中转，确认中转服务状态
4. 查看 DevTools Network 面板是否有请求发出
```

#### 图表不显示

```
原因：ECharts 实例初始化失败
排查：
1. 确认容器 DOM 元素已正确渲染
2. 检查 Console 是否有 ECharts 报错
3. 窗口缩放后图表可能需要 resize 事件触发
```

#### 打包失败

```
原因：electron-builder 配置或环境问题
排查：
1. 确认 package.json 中 build 配置正确
2. Windows 上需要管理员权限（NSIS 注册表操作）
3. 磁盘空间需 >= 2GB 剩余
4. 如签名失败，设置 CSC_IDENTITY_AUTO_DISCOVERY=false 跳过
```

## 八、安全运维

### 8.1 安全配置清单

| 项目 | 当前状态 | 说明 |
|------|----------|------|
| contextIsolation | ✅ 启用 | 渲染进程与主进程隔离 |
| nodeIntegration | ✅ 禁用 | 渲染进程无法直接访问 Node.js |
| IPC 输入校验 | ✅ Zod | 所有 IPC invoke 均经过 schema 验证 |
| API Key 存储 | 环境变量 | 不硬编码、不暴露到渲染进程 |
| Bridge 认证 | Token 可选 | 建议生产环境配置 Token |
| CORS | 仅 localhost | Bridge 仅接受本地请求 |
| 数据存储 | 纯本地 | 不向外部发送设备数据 |

### 8.2 安全建议

1. **API Key 保护**：切勿将 `.env` 文件提交到版本控制
2. **Bridge Token**：生产环境务必配置 `ELECTRON_BRIDGE_TOKEN`
3. **网络隔离**：如在工业内网部署，确保 Bridge 端口不暴露到外网
4. **定期更新**：关注 Electron 安全公告，及时更新框架版本
5. **最小权限**：应用无需管理员权限运行
